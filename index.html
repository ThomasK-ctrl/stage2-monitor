What is a Stage 2 Loan?
Under IFRS 9, a Stage 2 loan has experienced a significant increase in credit risk since origination but is not yet non-performing. The borrower is still paying, but credit quality is deteriorating. Banks must provision for lifetime expected credit losses on Stage 2 loans (vs 12-month ECL for Stage 1). Historically, elevated Stage 2 balances precede NPL increases by 6–18 months. A ratio above 10% warrants monitoring; above 12% signals elevated NPL supply risk.
What is a Stage 3 Loan (NPL)?
A Stage 3 loan is credit-impaired — the borrower is more than 90 days past due or the bank has objective evidence of impairment. This is the regulatory definition of a Non-Performing Loan (NPL) under IFRS 9 and EBA guidelines. Banks provision for lifetime ECL and full provisioning is required under the ECB's prudential backstop within 7–9 years (secured) or 3 years (unsecured). NPL ratio = Stage 3 / Total Gross Loans. EU/EEA aggregate: 1.84% at Jun 2025.

⟳ How to Update Data
Fresh data from EBA in one command
1
Open your Terminal (Mac: Cmd+Space → "Terminal") or Command Prompt (Windows: Win+R → "cmd")
2
Navigate to your project folder:
cd ~/Desktop/stage2-monitor
3
Run the update script:
python update_data.py
The script downloads the latest EBA data, updates data/stage2.json, and prints a summary.
4
Reload this page in your browser — the dashboard refreshes automatically.
5
To publish updates online, push to GitHub:
git add data/stage2.json
git commit -m "Update EBA data"
git push
Your live URL updates within 2 minutes.
✓ EBA publishes new Transparency Exercise data every November. Run update_data.py after each publication.
